# AEGIS(Automated Equity & Global Investment System) 개발 로드맵 v3.1
*버전: 로드맵 v1.0 | 최종 업데이트: (현재 날짜) | 담당: 프로젝트 관리자*

---

## **Phase 1: 안정화 (v3.0 → v3.1)**

### **목표 및 KPI**
- **목표:** 현재 버전의 안정성 확보, 성능 저하 요소 제거, 유지보수성 향상.
- **KPI:**
    - 버그 리포트 90% 이상 해결
    - 시스템 평균 응답 시간 40% 개선(현재 850ms → 500ms 이하)
    - 코드 커버리지 70% → 85% 이상
    - 핵심 모듈의 기술 부채 지수(SonarQube) 20% 감소

### **주요 기능 목록**
1.  **버그 수정:**
    - 백테스팅 중 간헐적 데이터 불일치 현상 수정
    - 특정 조건에서의 메모리 누수 패치
    - 알림 전송 실패 오류 해결
2.  **성능 최적화:**
    - 데이터베이스 쿼리 최적화 (인덱스 재설계, N+1 문제 해결)
    - 캐싱 레이어 도입 (Redis를 활용한 자주 조회되는 시장 데이터 캐싱)
    - API 응답 페이로드 최소화
3.  **코드 리팩토링:**
    - 모듈 간 강결합 의존성 해소 (Interface 도입)
    - 중복 코드 모듈화
    - 로깅 시스템 표준화 및 구조화

### **예상 기간**
- **총 기간:** 8주
- **세부:** 기획/분석(1주), 개발(5주), 테스트/안정화(2주)

### **필요 리소스**
- **인력:** 백엔드 개발자 3명, 프론트엔드 개발자 1명, QA 엔지니어 1명, DevOps 엔지니어 0.5명
- **기술:** Redis 클러스터, APM 도구 (Datadog), 프로파일링 도구

### **의존성 및 리스크**
- **의존성:** 모든 버그에 대한 명확한 재현 시나리오 확보.
- **리스크:**
    - 리팩토링 과정에서 새로운 비호환성 발생 가능.
    - **완화책:** 철저한 단위/통합 테스트와 단계적 배포(Canary Release) 수행.

### **테스트 전략**
- **단위 테스트:** 모든 리팩토링 모듈에 대한 단위 테스트 작성/보강.
- **통합 테스트:** 모듈 간 상호작용 및 API 엔드포인트 검증.
- **부하 테스트:** 최적화 후 성능 지표 측정 및 검증.
- **회귀 테스트:** 기존 기능에 영향이 없는지 전체 시나리오 테스트.

## **Phase 2: 기능 강화 (v3.1 → v3.2)**

### **목표 및 KPI**
- **목표:** 시스템의 예측 정확도와 사용성 향상, 의사결정 지원 강화.
- **KPI:**
    - AI 모델 예측 정확도 (F1-Score) 5%p 향상
    - 알림 도달률 및 확인률 95% 이상
    - 백테스팅 실행 속도 50% 개선
    - 사용자 피드백 기반 개선사항 10건 이상 반영

### **주요 기능 목록**
1.  **AI 모델 업그레이드:**
    - Transformer 기반 시계열 예측 모델 도입 실험
    - 앙상블 모델 구조 개선 및 하이퍼파라미터 자동 최적화 (Optuna)
2.  **실시간 알림 시스템:**
    - 다중 채널 지원 (Email, Slack, Telegram) 확대
    - 사용자 정의 알림 조건 설정 (기술적 지표, 뉴스 키워드)
    - 알림 중요도 분류 및 스누즈 기능
3.  **백테스터 개선:**
    - 커스텀 전략 빌더 UI 제공
    - 더 정교한 거래 비용 및 슬리피지 설정 옵션 추가
    - 백테스팅 리포트 시각화 개선 (Drawdown, Sharpe Ratio 등 심화 분석)

### **예상 기간**
- **총 기간:** 12주
- **세부:** R&D/설계(3주), 핵심 개발(6주), 통합/테스트(3주)

### **필요 리소스**
- **인력:** AI/ML 엔지니어 2명, 풀스택 개발자 2명, 프론트엔드 개발자 1명, QA 1명
- **기술:** 추가 GPU 리소스 (모델 학습), 메시징 큐 (RabbitMQ/Kafka), 시각화 라이브러리 고도화

### **의존성 및 리스크**
- **의존성:** Phase 1 안정화 완료. 고품질 학습 데이터 확보.
- **리스크:**
    - 새로운 AI 모델의 실시간 성능 저하 또는 불안정.
    - 알림 시스템 확장으로 인한 지연 발생.
    - **완화책:** A/B 테스트를 통한 모델 성능 비교, 알림 시스템에 메시지 큐 도입 및 부하 테스트.

### **테스트 전략**
- **A/B 테스트:** 새 AI 모델 vs 기존 모델 성능 비교 (Paper Trading 환경).
- **통합 테스트:** 알림 엔진과 백테스터, AI 모듈 간 연동 테스트.
- **사용성 테스트:** 백테스터 UI에 대한 사용자 그룹 테스트 진행.

## **Phase 3: 확장 (v3.2 → v3.5)**

### **목표 및 KPI**
- **목표:** 시장 및 사용자 기반 확대, 접근성 향상.
- **KPI:**
    - 지원 거래소 3개 → 6개 이상 (해외 주요 거래소 추가)
    - 동시 관리 가능 포트폴리오 1개 → 5개
    - 모바일 앱 DAU (Daily Active Users) 500명 달성
    - 글로벌 사용자 트래픽 30% 증가

### **주요 기능 목록**
1.  **해외 시장 지원:**
    - 해외 거래소 API 연동 (예: Coinbase, Kraken, Binance Global)
    - 다국어 (영어) 지원 및 지역별 규정 준수 체크
    - 외화 (USD, EUR) 기준 포트폴리오 평가
2.  **다중 포트폴리오:**
    - 사용자별 독립된 포트폴리오 생성 및 관리 기능
    - 포트폴리오 간 자산 할당 및 성과 비교 대시보드
3.  **모바일 대시보드:**
    - iOS/Android 하이브리드 앱 개발 (React Native)
    - 핵심 지표 모니터링, 알림 확인, 수동 주문 체결 기능
    - 생체 인증 (지문/얼굴)을 통한 보안 강화

### **예상 기간**
- **총 기간:** 16주
- **세부:** 해외 API 분석/설계(4주), 핵심 개발(8주), 모바일 개발 및 통합(4주)

### **필요 리소스**
- **인력:** 백엔드 개발자 2명, 모바일 개발자 2명, 프론트엔드 개발자 1명, 법무/규제 자문 0.5명
- **기술:** 해외 거래소 API 접근 권한, 모바일 앱 배포 인프라 (App Store, Play Console), 다국어 관리 시스템

### **의존성 및 리스크**
- **의존성:** Phase 2의 안정적인 AI/알림 인프라. 해외 거래소의 API 신청 및 승인.
- **리스크:**
    - 해외 거래소 API의 지연 및 안정성 문제.
    - 모바일 앱의 보안 취약점.
    - 각국 금융 규제 상의 문제.
    - **완화책:** API Fallback 메커니즘 구현, 모바일 보안 감사 수행, 법무팀과의 사전 협의.

### **테스트 전략**
- **연동 테스트:** 각 해외 거래소 API와의 엔드투엔드 테스트 (Sandbox 환경).
- **로컬라이제이션 테스트:** 다국어 및 통화 표시 테스트.
- **모바일 보안 테스트:** 정적/동적 애플리케이션 보안 테스트(SAST/DAST).
- **부하 테스트:** 다중 포트폴리오 생성 시 서버 성능 테스트.

## **Phase 4: 자동화 (v3.5 → v4.0)**

### **목표 및 KPI**
- **목표:** 투자 프로세스의 완전 자동화 및 지능형 리스크 관리 구현.
- **KPI:**
    - 자동 매매 실행률 (사용자 설정 대비) 99% 이상
    - 리스크 규칙 위반 시 자동 대응률 100%
    - 주요 리스크 지표 (Max Drawdown, VaR) 실시간 모니터링 및 경고
    - ML 파이프라인으로 인한 모델 개편 주기 4주 → 2주로 단축

### **주요 기능 목록**
1.  **완전 자동 매매:**
    - 사용자 승인 하에 AI 신호 기반 전액 자동 주문 실행
    - 스마트 주문 라우팅 (최적의 거래소 자동 선택)
    - 자동 재평가 및 재조정 스케줄러
2.  **리스크 관리 강화:**
    - 사용자 맞춤형 리스크 예산 설정 및 준수 감시
    - 시장 변동성 급등 시 자동 헤징 전략 실행
    - 블랙스왈 기반 포트폴리오 리스크 분석 대시보드
3.  **ML 파이프라인:**
    - 데이터 수집 → 전처리 → 학습 → 검증 → 배포의 완전 자동화 CI/CD 파이프라인 구축
    - 모델 성능 지속적 모니터링 및 자동 롤백 기능

### **예상 기간**
- **총 기간:** 20주
- **세부:** 거래 실행 엔진 설계(5주), 리스크 시스템 개발(5주), ML 파이프라인 구축(6주), 통합 및 규제 준비(4주)

### **필요 리소스**
- **인력:** 시스템/트레이딩 엔지니어 2명, AI/ML 엔지니어 2명, 백엔드 개발자 2명, 금융 리스크 전문가 1명, 법무 1명
- **기술:** 고가용성 거래 실행 서버,低지연 네트워크, MLFlow/Kubeflow 같은 MLOps 플랫폼, 감사 로그 시스템

### **의존성 및 리스크**
- **의존성:** Phase 3의 확장된 시장 인프라. 금융 당국의 자동매매 관련 규제 완벽 이해 및 준수.
- **리스크:**
    - **가장 높음:** 자동 매매 로직 오류로 인한 재무적 손실.
    - 시스템 장애로 인한 거래 기회 상실.
    - ML 모델의 디그레이데이션(성능 저하).
    - **완화책:** 제한된 자본으로의 시범 운영(Pilot), 매매 전략 사전 시뮬레이션 의무화, 매수/매도 한도 설정, 수동 오버라이드 버튼 필수 장착. 모델 성능 지속적 모니터링.

### **테스트 전략**
- **시뮬레이션 테스트 (Paper Trading):** 자동 매매 전략을 수개월간 가상 자본으로 철저히 검증.
- **감사 추적 테스트:** 모든 자동 거래에 대한 불변의 감사 로그 생성 확인.
- **재해 복구 테스트:** 시스템 장애 시 자동 매매 중단 및 안전 모드 진입 테스트.
- **스트레스 테스트:** 극단적 시장 상황(Flash Crash)에서의 리스크 시스템 반응 테스트.
